You are a clinical trial data extraction engine. Extract structured data from the clinical trial listing document below and return ONLY a JSON object.

### OUTPUT RULES (STRICT)
- Output MUST be valid JSON only
- Do NOT include any explanatory text
- Do NOT include markdown code fences
- Output must start with { and end with }
- Every value must be a string
- If a value is missing, use empty string ""

### WHAT TO EXTRACT
- listing_number: the number after the word "Listing" e.g. 16.2.1.1
- title: the descriptive heading of the listing
- columns: the list of column headers (the row that starts with "Subject ID")
- rows: all data rows below the header, each as a dict mapped to column names

### MULTI-LINE COLUMN HEADERS
Some tables have stacked/multi-line column headers where one logical column
is described across 2-3 lines. In this case:
- Combine the lines into one clean column name
- Example: "Actual" on line 1 and "Treatment" on line 2 = "Actual Treatment"
- Do NOT use | as a separator in column names
- The correct columns for this document are the logical data fields,
  not the raw text lines

### SUBJECT INFO IN FIRST ROW
For study drug administration listings, the first row of each subject group 
contains stacked info: Subject ID on line 1, then "Age, Sex, Race" on line 2.
Extract these into SEPARATE columns:
- "Subject ID": just the ID number e.g. "10011002"  
- "Age (y)": just the age number e.g. "53"
- "Sex": just M or F
- "Race": the race abbreviation or full name

Do NOT combine Age, Sex, Race into one field.

### WHAT TO SKIP
- Footer lines: abbreviations, dates, page numbers, file paths, notes
- Separator lines made of dashes or equals signs
- Repeated header rows

### OUTPUT FORMAT
{"listing_number":"","title":"","columns":[],"rows":[]}
